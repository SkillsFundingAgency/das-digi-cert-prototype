    • Notable changes from the v6 Figma prototype include:
      • Removing QR code functionality

      During this iteration, I looked at different ways to implement the 'Check your answers' shortcut in this prototype. I've ended up with code for two different versions, which are stored in the following file in a new 'code snippets' folder:

      • If statements for authorisation pathing

      Version 1 does one big check. If the user answered all of the upcoming questions already, they skip to CYA. If they haven't, they go to the next question as normal.

      Version 2 checks each upcoming question in order. If it's not been answered yet, the user goes to its question page. If it has been answered, it gets skipped. If all questions have been answered you skip ahead to the CYA page.

      I also have a theoritcal Version 3 in mind. This version would still check based on whether all of the upcoming questions have been answered, but rather than changing the actino of the 'Submit' button, it would instead add an additional button that takes the user to the CYA page. This would allow the user to more quickly change multiple answers, while keeping the ability to skip through to the CYA page when appropriate.
